## 2026-06-11

### Orca 项目重构 - Chat 界面优化

**完成的工作：**

1. **右侧可折叠侧边栏** — 新增三个标签页：
   - **任务列表**：从行内移除，独立展示任务进度条和各任务状态（待处理/执行中/已完成）
   - **已修改文件**：自动追踪工具操作中修改的文件，显示文件路径、操作类型和时间
   - **Git 状态**：显示当前分支、已修改/未跟踪文件数、最近提交信息，提供"查看改动"和"提交建议"快捷按钮
   - 支持拖拽调整宽度（240-500px），状态持久化到 localStorage

2. **上下文窗口指示器** — 在输入框上方添加 Token 用量进度条：
   - 实时估算当前对话的 Token 使用量
   - 根据模型自动匹配上下文窗口上限
   - 颜色分级：绿色(<60%) / 黄色(60-85%) / 红色(>85%)
   - 超 85% 时显示警告提示

3. **流式渲染性能优化**：
   - 使用 `requestAnimationFrame` 批量更新 state，减少高频 SSE 流造成的渲染抖动
   - `localStorage` 去抖写入（2 秒间隔）
   - 连接断开重试时自动 flush 缓冲区

4. **后端新增**：
   - `POST /api/git/status` — 查询工作区 Git 状态（分支、改动、未跟踪文件、最近提交）

### 第二轮：架构审计 & Bug 修复

5. **ECharts 内存泄漏修复** — Dashboard 的两个图表 useEffect 添加 `myChart?.dispose()` cleanup，防止每次重渲染时创建新实例而不销毁旧实例

6. **`scanApps()` 异步化** — 使用 `setImmediate` 延迟执行 + stale-while-revalidate 缓存策略，避免 `execSync("tasklist")` 阻塞 Express 事件循环。新增 `POST /api/apps/refresh` 端点用于强制刷新

7. **ErrorBoundary 主题适配** — 从硬编码深色主题改为根据 `document.documentElement.classList` 动态适配亮/暗主题，添加 `componentDidCatch` 日志输出

8. **TypeScript 类型修复** — 修复 git status 中隐式 `any` 类型的 filter 回调参数

### 第三轮：全面 Bug 修复

9. **修复内存泄漏** — `executeAgentCompletions` 递归调用中 `res.on("close")` 会重复注册监听器。修复方式：使用 `res` 属性标记 + 单次注册，`__orca_closeListenersAdded` / `__orca_clientDisconnected` 在所有递归层级共享状态。

### 第四轮：全面改进实施

15. **构建产物优化** — `package.json` 中 esbuild 命令添加 `--minify` 参数，减小 bundle 体积 (~30%)。

16. **缓存策略增强** — `cache.ts` 新增：
   - 7天 TTL 自动过期
   - 1000条 LRU 淘汰上限
   - `lastAccessed` 追踪字段 + 旧格式自动升级
   - 新增 `POST /api/cache/purge` 和 `GET /api/cache/stats` 管理端点

17. **日志轮转** — `index.ts` 新增：
   - 10MB 大小阈值，5 份备份文件
   - 启动时检查 + 运行时每 ~100 行概率检查
   - 流水号滚动 (`orca.log` → `orca.log.1` → ... → `orca.log.5`)

18. **健康检查增强** — `GET /health` 返回：内存使用(MB)、PID、平台、Node 版本、MCP 服务状态、缓存统计。

19. **路径遍历防护增强** — 文件操作工具新增：
   - 空字节注入检测
   - 绝对路径拒绝
   - 双重路径验证（规范化路径 + symlink 解析路径）
   - 统一 `validateFilePath` / `resolveSafePath` 函数

20. **深色/浅色主题适配** — 前端改进：
   - `index.css` 新增 `--color-error`、`--color-error-bg`、`--color-code-bg`、`--color-code-fg` 变量
   - `App.tsx` ErrorBoundary 移除 `isDark` 三元检查，改用 CSS 变量
   - `Dashboard.tsx` 新增 `readThemeColors()` 辅助函数，ECharts 配置统一使用 CSS 变量读取
   - 补全暗色主题对应值

21. **index.ts 模块化拆分（核心改进）** — 从 3433 行精简至 2487 行：
   - `src/services/tools.ts` (417行) — `handleAgentToolCall` + 路径验证
   - `src/services/skills.ts` (358行) — 技能管理、脚本运行、终端命令
   - `src/services/billing.ts` (192行) — 计费统计、模型定价
   - `src/services/helpers.ts` — 共享 log 函数
   - index.ts 保留 Express 路由、Agent 循环、App 扫描等核心逻辑

12. **修复 mcp.ts readline 资源泄漏** — `MCPClient.kill()` 未关闭 `readline.Interface`。添加 `reader` 实例变量，`kill()` 中调用 `reader.close()`。

13. **修复 main.js 端口占用提示** — `dialog.showMessageBox` message 字段缺少实际端口号。改为 `` '端口 ' + PORT + ' 已被其他程序占用' ``。

14. **修复 `scanApps` execSync 超时缺失** — 7 个 `where` 命令调用无 timeout 参数。全部添加 `timeout: 5000`。
