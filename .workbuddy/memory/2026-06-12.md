## 2026-06-12

### 第五轮：剩余四项改进实施

22. **Discover SSE 流式同步** — 新增两个端点：
   - `GET /api/discover-all` — SSE 流式发现全部已配置供应商的模型列表，实时推送进度
   - `POST /api/discover-sync` — 批量将发现的模型写入供应商配置
   - 支持 `init → progress → result → done` 事件流

23. **对话历史导出/导入** — Chat.tsx 新增：
   - `handleExportMarkdown` — 导出当前对话为 Markdown 文件（含模型、日期、角色标记）
   - `handleExportJSON` — 导出全部对话为 JSON 文件（可跨设备恢复）
   - `handleImportJSON` — 导入 JSON 文件到当前会话列表
   - 侧边栏 "新建会话" 按钮下方新增 MD / JSON / 导入 三个按钮

24. **Skills 市场集成** — 新增 `POST /api/skills/github-import`：
   - 输入 GitHub repo URL，自动解析 owner/repo/branch/subPath
   - 通过 GitHub Contents API 递归下载所有文件
   - 自动检测 SKILL.md、跳过二进制文件、去重 skillId
   - 零外部依赖（纯 fetch + fs）

26. **启动测试** — 构建并启动 Electron 桌面应用：
   - TypeScript 编译通过 → `esbuild` 打包 `dist/bundle.js` (926.7KB)
   - 前端 Vite 构建耗时 5s
   - 发现启动失败根因：环境变量 `ELECTRON_RUN_AS_NODE=1` 导致 Electron 以 Node.js 模式运行
   - 取消该变量后 `electron .` 正常启动，服务监听 `127.0.0.1:18080`，技能初始化成功
   - 通过 `tsc --noEmit` 编译验证
   - 运行方式：`npx ts-node src/tests/transform.test.ts`
